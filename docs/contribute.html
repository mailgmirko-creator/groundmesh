Write words → see code

Anyone can help GroundMesh. In 60 seconds you can open a feature request that we’ll translate into tasks and code.

➕ New Feature / Improvement 📖 Quickstart (Markdown)
Brave-friendly · No trackers · Human-readable first

How to help (in 4 steps)

  • Open New Issue → Feature / Improvement
  • Describe Purpose, User Story, Inputs/Outputs
  • Add 2–3 Acceptance Criteria
  • Submit — we turn it into a PR

What happens next?

We convert your words into a small task list, generate code/UX, and open a pull request for you to review.

You’ll see: OK: proposals/<your-spec>.md in our validator.

Prefer a file instead of an Issue?

Copy the template: proposals/spec-template.md, write your spec, and open a PR.

CI will gently warn if key sections are missing.

Tip: works even on slow connections, with privacy tools enabled. If something feels rough, open a quick issue and tell us.
