# GroundMesh New Chat Handoff

Use this when starting a new chat that does not already know the current GroundMesh state.

## How to use it
1. Open this file.
2. Paste the short handoff block into the new chat.
3. Add one sentence about what you want next.

Example:
`Read this handoff, then continue with TSL and Balance Engine ideas.`

## Paste block
I am continuing GroundMesh from an existing Codex/GitHub working lane.

Important context:
- GroundMesh is the canonical core repo and GitHub is the shared memory spine.
- Work style is Atlas-first, small guarded batches, less but more anchored.
- Public docs surface is live and coherent.
- Registration is not broadly open; it is an invited-circle pilot only.
- The first self-registration has already been completed locally.
- Local pilot tooling now exists for start/stop/open flows under `scripts/` and `tools/`.
- Decisions already anchored include the ChatGPT/Codex/GitHub operating pattern, the ACTIVE/DEV promotion model, and registration-as-pilot.
- The next topic I want to work on is: [replace this line with the actual next topic].

Please inspect current repo state first, summarize the smallest relevant context, and then propose or take the next smallest real step.

## Why this exists
This keeps new chats from drifting into long recap mode.
It is not a full memory dump. It is a compact reload key.
